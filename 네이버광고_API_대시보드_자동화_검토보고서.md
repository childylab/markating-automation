# 네이버 광고 데이터 API 연동 및 대시보드 자동화 검토 보고서

**작성일:** 2026년 7월 7일  
**목적:** 네이버 광고(SA, DA, 애드부스트) 성과 데이터의 API 기반 자동 수집 및 대시보드 구축 가능 여부 확인

---

## 1. 요약

네이버 광고 자동화 검토 결과, **SA와 DA의 데이터 수집 방식을 이원화**해야 한다.

- **SA (검색광고)** → 공식 API(`api.searchad.naver.com`)로 완벽하게 자동화 가능. 테스트 완료.
- **DA (성과형 디스플레이/애드부스트)** → API가 존재하나, **네이버 공식 파트너사에만 제공**. 일반 광고주는 접근 불가. 브라우저 자동화(Playwright)로 보고서 자동 다운로드 방식 채택.

## 2. API 구조 확인

### 2.1 단일 통합 API

| 항목 | 내용 |
|------|------|
| API 엔드포인트 | `https://api.searchad.naver.com` |
| 공식 문서 | https://naver.github.io/searchad-apidoc/ |
| GitHub 저장소 | https://github.com/naver/searchad-apidoc |
| 샘플 코드 | Python, Java, PHP, C# 제공 |

"검색광고 API"라는 명칭을 사용하고 있으나, 실제로는 네이버 광고 관리 시스템(`manage.searchad.naver.com`)에서 운영되는 **모든 광고 상품**의 데이터를 커버하는 통합 API이다.

### 2.2 SA / DA / 애드부스트 통합 운영 근거

- 네이버 광고 관리 시스템이 SA, DA(성과형 디스플레이), 애드부스트를 하나의 플랫폼에서 통합 관리
- API 인증 체계(API Key + Secret Key + Customer ID)가 동일
- 캠페인 ID 기반으로 성과 데이터를 조회하는 구조이므로, 캠페인 유형(SA/DA/애드부스트)과 무관하게 동일 API로 접근 가능
- Dataddo, Adriel 등 서드파티 마케팅 데이터 플랫폼이 동일 API 인증 정보로 Naver 광고 전체 데이터를 연동 중
- 공식 GitHub 계정(`naver-searchad`)에서 마스터 리포트 스펙 문서(`master-reports-spec.md`)를 2025년 6월 갱신 — DA 포함 리포트 필드 정의

### 3.2 운영 현황

| 계정 | Customer ID | 유형 | 운영 주체 | 주요 내용 |
|------|-------------|------|-----------|-----------|

- DA 데이터 조회: `X-Customer: 1667290`
- SA 데이터 조회: `X-Customer: 1667291`

## 5. 주요 API 엔드포인트 및 활용 범위

| 용도 | 엔드포인트 | 설명 |
|------|-----------|------|
| 캠페인 목록 조회 | `GET /ncc/campaigns` | SA/DA/애드부스트 포함 전체 캠페인 |
| 광고그룹 조회 | `GET /ncc/adgroups` | 캠페인 내 광고그룹 |
| 키워드 조회 | `GET /ncc/keywords` | SA 키워드 단위 |
| 소재 조회 | `GET /ncc/ads` | 광고 소재 |
| 성과 요약 통계 | `GET /stats` | 지정 ID의 성과 데이터 |
| 마스터 리포트 | `GET /stat-reports` | 대량 성과 데이터 다운로드 |

### 5.1 조회 가능한 성과 지표 (fields)

**stat API (요약 통계)**
```
impCnt      — 노출수
clkCnt      — 클릭수
salesAmt    — 광고비(소진액)
ctr         — 클릭률
cpc         — 클릭당 비용
ccnt        — 전환수 (모든 전환 합산: 구매+장바구니+기타)
convAmt     — 전환매출 (모든 전환 합산)
crto        — 전환율
viewCnt     — 조회전환수
viewAmt     — 조회전환매출
avgRnk      — 평균 노출 순위
```

> ⚠️ stat API의 `ccnt`/`convAmt`는 구매, 장바구니, 회원가입 등 **모든 전환이 합산**되어 ROAS가 과대 산출됨.

**stat-reports API (마스터 리포트 — 전환 상세)**

| 리포트 타입 | 설명 |
|------------|------|
| `AD_CONVERSION_DETAIL` | 소재 단위 전환 상세 (전환유형별 분리) |
| `CRITERION_CONVERSION` | 키워드/타겟 단위 전환 상세 |

`AD_CONVERSION_DETAIL` 리포트 컬럼 (탭 구분):
```
날짜 | CustomerID | 캠페인ID | 광고그룹ID | 키워드ID | 소재ID | 채널ID | 시간대 | 전환경로코드 | 매체ID | 디바이스 | 전환경로구분 | 전환유형 | 전환수 | 전환금액
```

전환유형 값:
- `purchase` — **실제 구매** (정확한 ROAS 산출용)
- `add_to_cart` — 장바구니 추가
- 기타 (회원가입, 신청 등)

**→ `purchase` 전환만 필터링하면 정확한 구매 ROAS 계산 가능**

마스터 리포트를 통해 이보다 더 상세한 필드(전환값, ROAS, 디바이스별 분리 등)를 일별로 추출할 수 있다.

## 6. 기술적 실현 가능성

### 6.1 데이터 수집

- Python `requests` 라이브러리로 REST API 호출
- HMAC-SHA256 서명 생성 후 헤더에 포함
- Customer ID 2개를 순회하며 데이터 수집
- 일별 배치 또는 4시간 주기 수집 가능

### 6.2 대시보드 구축 옵션

| 방식 | 장점 | 단점 |
|------|------|------|
| Google Sheets + Apps Script | 무료, 공유 쉬움 | 대량 데이터 한계 |
| Google Looker Studio | 시각화 강력, 공유 편리 | Sheets 연동 필요 |
| 자체 웹 대시보드 (Streamlit 등) | 커스터마이징 자유 | 서버 운영 필요 |
| Notion DB + 자동화 | 팀 협업 용이 | 실시간성 부족 |

### 6.3 자동화 실행 환경

- **AWS Lambda / GCP Cloud Functions** — 서버리스, 스케줄 실행
- **GitHub Actions** — 무료 범위 내 일별 수집 가능
- **로컬 cron** — 가장 간단, 안정성 낮음

## 8. 최종 자동화 아키텍처

### 8.1 전체 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    대시보드 (최종 출력)                        │
│              HTML/CSS/JS (Netlify 배포)                       │
│              + 로컬 Flask 서버 (API 프록시)                    │
└─────────────────────┬───────────────────────────────────────┘
                      │
          ┌───────────┴───────────┐
          │                       │
    ┌─────┴─────┐          ┌─────┴──────┐
    │  SA 데이터  │          │  DA 데이터   │
    │  (API)     │          │ (수동 CSV   │
    │            │          │  업로드)     │
    └─────┬─────┘          └─────┬──────┘
          │                       │
  네이버 검색광고 API      ads.naver.com에서
  api.searchad.naver.com   수동 보고서 다운로드
```

### 8.2 SA 수집 — API 방식 ✅

| 항목 | 내용 |
|------|------|
| 방식 | REST API 호출 (Python requests) |
| 엔드포인트 | `https://api.searchad.naver.com` |
| 인증 | API Key + Secret + HMAC-SHA256 서명 |
| 데이터 | 캠페인/광고그룹/키워드 단위 성과 + 구매전환 분리 |
| 갱신 주기 | 실시간 (대시보드 버튼 클릭) |
| 안정성 | 높음 (공식 API) |

### 8.3 DA 수집 — 수동 CSV 업로드

| 항목 | 내용 |
|------|------|
| 방식 | ads.naver.com에서 수동으로 보고서 CSV 다운로드 → 대시보드에 업로드 |
| 이유 | DA API는 공식 파트너사 전용. Playwright 자동화는 네이버 봇감지로 불가. |
| 데이터 | 캠페인/광고그룹 단위 성과 (CSV) |
| 갱신 주기 | 수동 (필요 시) |

### 8.4 DA Playwright 자동화 시도 — ❌ 실패 사례

Playwright를 통한 DA 보고서 자동 다운로드를 시도했으나 실패:

1. **새 브라우저 프로필** → 네이버가 봇으로 감지, 무한 추가 인증(영수증 인증) 요구
2. **실제 크롬 프로필 사용** → 크롬이 이미 열려있으면 프로필 잠금 충돌
3. **크롬 프로필 복사본** → about:blank에서 멈춤
4. **Playwright 전용 persistent context** → 로그인 시도 중 봇감지, 페이지 조작 없이 대기해도 네이버가 자동화 도구를 감지하여 정상 로그인 불가
5. **`--disable-blink-features=AutomationControlled` 플래그** → 효과 없음

**결론:** 네이버는 Playwright/Selenium 등 브라우저 자동화 도구를 적극적으로 감지·차단하고 있어 DA 데이터 자동 수집은 현재 불가능.

### 8.5 오케스트레이션 — n8n 활용

| 역할 | 구현 방식 |
|------|-----------|
| SA 데이터 수집 스케줄링 | n8n Cron 트리거 → Flask 서버 API 호출 |
| DA 데이터 | 수동 CSV 업로드 (자동화 불가) |
| 데이터 정제 및 적재 | n8n 내 Function 노드 |
| 대시보드 갱신 | SA는 자동, DA는 수동 |
| 알림 | n8n → Slack/이메일 (에러 발생 시) |
