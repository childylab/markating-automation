# 마케팅 퍼포먼스 대시보드 — 기획 방향

---

## 1. 목적

마케터가 1차로 정리한 스프레드시트나 Baserow 테이블을 가공하여 대시보드를 만드는 방식이 아닌, **각 광고 매체를 직접 연동**하여 왜곡이나 수동 유지보수가 개입되지 않는 **순수 지표 기반 대시보드**를 구축한다.

아드리엘 등 서드파티 마케팅 대시보드 툴과 동일한 방향이되, 자체 구축을 통해 커스터마이징과 비용 통제가 가능한 구조를 만든다.

---

## 2. 매체 기반 분류

대시보드의 1차 분류 단위는 **매체(채널)**로 한다.

| 분류 | 매체 | 상품 예시 |
|------|------|----------|
| 네이버 SA | 검색광고 | 브랜드검색, 쇼핑검색, 파워링크 |
| 네이버 DA | 성과형 디스플레이 | GFA, 애드부스트, 스마트채널 |
| 메타 | Meta Ads | Facebook, Instagram 광고 |
| 구글 | Google Ads | 검색, 디스플레이, 유튜브, P-Max |
| 기타 | 추가 매체 | 카카오모먼트, 틱톡 등 |

---

## 3. 매체 확대 로드맵

우선 네이버 SA/DA로 시작하고, 점차 API를 제공하는 모든 매체로 확대한다.

- **Phase 1 (현재):** 네이버 SA (API 직연동) + 네이버 DA (CSV 업로드)
- **Phase 2:** Meta Ads API 연동, Google Ads API 연동
- **Phase 3:** SNS 채널 지표 (인스타그램 인사이트, 유튜브 Analytics 등)
- **Phase 4:** 커머스 데이터 연동 (스마트스토어 주문 API 등 — 실매출 기반 ROAS)

---

## 4. 내부 저장소 및 서버 구조

매번 API를 호출해 데이터를 로드하지 않도록, **내부 저장소(DB)에 수집된 데이터를 적재**하고 대시보드는 저장소에서 읽어오는 구조를 갖춘다.

### 4.1 구조

```
[광고 매체 API] → [수집 서버] → [내부 DB] → [대시보드 프론트]
```

- **수집 서버:** 네이버 검색광고 API 등과 통신하여 데이터를 주기적으로 가져옴
- **내부 DB:** 수집된 일별 성과 데이터를 캠페인/날짜 단위로 저장
- **대시보드:** DB에서 조회하여 즉시 표시 (매번 API 호출 불필요)

### 4.2 DA CSV 업로드 시 중복 처리

DA(성과형 디스플레이)는 API 직연동이 불가하여 CSV 업로드 방식을 사용한다.  
동일 캠페인 + 동일 날짜에 기존 데이터가 존재하는 경우, 업로드 시 분기 처리가 필요하다:

- **덮어쓰기:** 동일 캠페인/날짜 데이터를 새 CSV 값으로 대체 (기본 동작)
- **건너뛰기:** 이미 존재하는 날짜는 업로드하지 않고 신규 날짜만 추가

→ 업로드 시 사용자에게 선택지를 제공하거나, 기본값(덮어쓰기)을 설정해두고 옵션으로 변경 가능하게 한다.

---

## 5. 엑셀 내려받기

현재 조회 중인 뷰(기간 필터, 매체 필터 적용 상태)를 기준으로 **엑셀/CSV 파일로 내려받기** 기능을 제공한다.

- 캠페인 합산 테이블 + 일별 상세 데이터를 시트 분리하여 제공
- 파일명에 매체명 + 기간 자동 포함 (예: `SA_검색광고_2026-06-07_2026-07-07.xlsx`)

---

## 6. 시각화 (차트)

캠페인별 성과 추이를 시각적으로 파악할 수 있도록 차트를 제공한다.

- **꺾은선 그래프:** 일별 추이 (광고비, 구매매출, ROAS 등)
- **막대 그래프:** 캠페인 간 비교 (광고비, 전환수 등)
- 차트는 기간 필터와 연동되어 선택된 기간의 데이터로 표시

## 요약

| 항목 | 방향 |
|------|------|
| 데이터 소스 | 매체 직연동 (API 우선, 불가 시 CSV) |
| 분류 체계 | 매체 기반 (SA, DA, 메타, 구글 등) |
| 저장 | 내부 DB 적재 (매번 로드 불필요) |
| 중복 처리 | 덮어쓰기/건너뛰기 선택 |
| 내보내기 | 현재 뷰 기준 엑셀 다운로드 |
| 시각화 | 일별 추이 꺾은선 + 캠페인 비교 막대 차트 |
| 확장 | 점진적 매체 추가 (API 제공 매체 우선) |
